# Post-Processing Utilities for ONT mtDNA DualCall Pipeline

## Overview

This directory contains **post-processing utilities** designed to operate on the outputs generated by the main
ONT DualCall SNP and INDEL pipeline.

These tools are **downstream of variant calling**. They do **not** alter read alignments or variant evidence.
Their purpose is to:

- Organize per-sample outputs into consolidated folders
- Restore variant coordinates after alignment to a linearized mitochondrial reference
- Normalize BAM contig names for compatibility with IGV
- Prepare finalized files for review and visualization

All scripts assume that the main pipeline has completed successfully.

---

## Recommended Execution Order

The tools in this directory are intended to be used in the following order:

1. **Collect raw VCFs and BAMs**
   - `collect_vcfs_and_sorted_bams.sh`

2. **Correct VCF coordinates derived from a linearized reference**
   - `MITO_VCF_LINEAR_CORRECTOR.py`

3. **Clean BAM contig names for IGV compatibility**
   - `BAM_cleaner3_GUI.py`

4. **Collect finalized, corrected files for viewing**
   - `collect_corrected_vcfs_and_cleaned_bams.sh`

Each step produces outputs that may be used independently, depending on the analysis requirements.

---

## Script Descriptions

---

## 1. collect_vcfs_and_sorted_bams.sh

### Purpose

Collects per-sample VCF files and sorted BAM files from individual pipeline output directories
and copies them into centralized folders for batch analysis or inspection.

This script performs **file organization only** and does not modify file contents.

### Inputs

- Root output directory produced by the main pipeline

### Outputs

- Folder containing collected VCF files
- Folder containing collected sorted BAM files

### Usage

```bash
./collect_vcfs_and_sorted_bams.sh <pipeline_output_directory>
```

---

## 2. MITO_VCF_LINEAR_CORRECTOR.py

### Purpose

Corrects variant coordinates in VCF files generated using a **linearized mitochondrial DNA reference**,
restoring positions to the coordinate system of the original reference genome.

This script applies a deterministic position mapping derived from:
- The original (circular) mtDNA reference FASTA
- The linearized mtDNA reference FASTA

No variant alleles, genotypes, or quality values are modified.

### Inputs

- One or more VCF files generated against a linearized mtDNA reference
- Original mtDNA reference FASTA
- Linearized mtDNA reference FASTA

### Outputs

- Corrected VCF files with coordinates restored to the original reference

### Usage

```bash
python MITO_VCF_LINEAR_CORRECTOR.py
```

The script provides a graphical interface for selecting input files.

### Notes

- Intended only for VCFs produced using the corresponding linearized reference
- Reference FASTA files must match those used during alignment and calling

---

## 3. BAM_cleaner3_GUI.py

### Purpose

Cleans BAM file headers and contig names to improve compatibility with genome browsers such as IGV.

Typical operations include:
- Removing `_linearized` suffixes from contig names
- Removing reference version suffixes (e.g. `.1`)
- Regenerating BAM headers using `samtools`

The script does not modify read alignments or alignment coordinates.

### Inputs

- Folder containing BAM files

### Outputs

- Cleaned BAM files suitable for visualization
- Corresponding BAM index files

### Usage

```bash
python BAM_cleaner3_GUI.py
```

A graphical interface is provided for selecting input folders.

### Requirements

- `samtools`
- Python with `biopython`
- `tkinter` (for GUI)

---

## 4. collect_corrected_vcfs_and_cleaned_bams.sh

### Purpose

Collects finalized VCF and BAM files after coordinate correction and BAM cleaning,
placing them into centralized folders for final review and visualization.

This script is typically run after:
- VCF coordinate correction
- BAM contig name cleaning

### Inputs

- Directory containing corrected VCF files
- Directory containing cleaned BAM files

### Outputs

- Centralized folder containing final VCF files
- Centralized folder containing final BAM files

### Usage

```bash
./collect_corrected_vcfs_and_cleaned_bams.sh <source_directory>
```

---

## Design Principles

- Post-processing only; no variant calling or read filtering
- Deterministic and reproducible transformations
- Clear separation between analysis, correction, and visualization
- Suitable for audit, validation, and forensic workflows

---

## Notes and Caveats

- Coordinate correction assumes a single, known linearization strategy
- BAM cleaning affects headers only; downstream tools should be informed of reference naming
- Users are responsible for ensuring reference consistency across all steps
